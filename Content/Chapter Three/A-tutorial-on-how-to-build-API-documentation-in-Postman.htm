A tutorial on how to build API documentation in Postman

Below you will find a quick tutorial on how to build a simple documentation in the Postman tool.

Step 1: Install Postman

The first step is to install the Postman tool, which is a platform for testing and documenting APIs. You can download Postman from the official site at https://www.postman.com/downloads/ and install it on your computer.

When you open Postman you should have a view like this:

Step 2: Create a new project

So as to create a new project click on the "New" button on the navigation bar and select "API Documentation". Postman will take you to “Create a new collection” button. Click on the orange button ”Next” and then name your collection which means that you give a name to your API. I decided to call it simply an “API”.

Step 3: Add resources to the project

In Postman, you'll see the screen divided into three sections: “My Workspace”, “Overview” which is your main content area for personal work and “Documentation”. “My Workspace” includes the navigation bar and sidebars. Here you may also “Add a request” so as to add requests to your API. See the picture below:

Why don’t you try to create your own simple API? Click on “Add a request”. You will see a “New request” with some with fields to be filled in:

New Request

A drop down that here starts with “GET”

an URL

Fill in those green ovals with the following information:

New request: Get Users

Method: GET

URL: https://api.example.com/users

Now your screen should look like this and that would be your first endpoint - Get Users.

The field “Description” is your place to write short and precise information on what an endpoint does, i.e. Create a new user, Remove a user with the following number.

When you want to add another request place your coursor in the project “API” and select three dots. A drop down will will appear. Choose “add requests” and make other endpoints. T

Step 4: Generate your API documentation

This simple API was necessary to see how our API documentation would look like. Once again, place your coursor in the project “API” and select three dots. A drop down will will appear. This time choose “View documentation” and click on it. Postman will show yoy a nicely looking, neat and ordered documentation.

You may also publish API documentation on the Postman platform. Here's how to do it:

Right-click on the project name in the navigation panel and select "Publish Documentation".

Select "Postman" as the documentation format.

Enter the name of the documentation, select the publishing options and click "Publish."

View the API documentation.

Once you publish your API documentation, you will receive a unique link that you can share with others. Open this link in your web browser to view the generated API documentation in Postman.
